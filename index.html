接口

接口(interface): 用来描述类具有什么功能的技术，但是不给出每个功能的具体实现。

一个类只能继承(extends)一个父类，但是可以实现(implement)多个接口，并在需要接口的地方，随时使用使用实现了相应接口的对象。

1.1 接口概念

接口不是类，而是对类的一组需求进行描述，这些类遵循从接口描述的统一格式进行定义。

例如：Arrays类中的 sort 方法承诺可以对对象数组进行排序，但要求满足下列前提：对象所属类必须实现了Comparable接口。

提供实例域和方法实现的任务应该有实现接口的那个类来完成。因此，可以将接口看成是没有实例域的抽象类。

1.2 接口的特性

接口不是类，因此不能用 new 运算符实例化一个接口；但是我们却可以声明接口变量：

Comparable x; // OK

接口变量必须引用了实现接口的类对象：

x = New Employee(...); // OK provided Employee implements Comparble

可以用 instance 检查一个对象是否实现了某个特定的接口：

if(anObject instance of Comparble){...}

1.3 接口与抽象类

为什么Java要设计接口这样一个概念，而不是直接将Comparable设计为一个抽象类呢？

因为在Java中每个类只能继承一个类，但是可以实现多个接口。

1.4 静态方法

接口中可以增加静态方法，但是这样有违于将接口作为抽象规范的初衷，因此不建议这样做。

1.5 动态方法

可以为接口实现一个默认实现，但是必须用default修饰符标记这样一个方法。

在Java SE 8中，可以把所有方法都声明为默认方法，这些默认方法什么也不做，也可以实现方法。

1.6 解决默认方法冲突

如果在一个接口中将一个方法定义为默认方法，在超类中或另一个接口中也定义了同样的方法，Java的处理规则如下：

  1. 超类优先，接口中同名的方法会被忽略；
  2. 接口冲突，如果一个超接口提供了一个默认方法，另一个接口提供了一个同名而且参数类型也相同的方法，必须覆盖着个方法解决冲突；
