回调(callback)指某个特定事件发生时应该采取的动作。
java.swing包中有一个Timer类，可以在在到达给定的时间间隔发出通知。定时器需要调用的方法需要传递对象所属的类实现了java.swing包中的ActionListener接口：

javax.swing.JOptionPane 1.2

  • static void showMessageDialog(Component parent, Object message)显示一个包含一条消息和 OK 按钮的对话框。 这个对话框将位于其 parent 组件的中央。如果parent为mull, 对话框将显示在屏幕的中央.

javax.swing.Timer 1.2

  • Timer(int interval, ActionListener listener)
    构造一个定时器，每隔interval毫秒通告 listener一次.
  • void start()
    启动定时器一旦启动成功，定时器将调用监听器的actionPerformed。
  • void stop()
    停止定时器。一旦停止成功，定时器将不再调用监听器actionPerformed。

java.awt.Toolkit 1.0

  • static Toolkit getDefaultToolkit()
    获得默认的工具箱。工具箱包含有关GUI环境的信息。
  • void beep()
    发出一声响铃。

对象克隆

我们知道，将一个初始变量直接赋给新变量，原变量和新变量都是对同一个对象的引用：任何一个变量的改变都会引起另一个变量变化（如图1所示）。

如果想复制一个新对象，它的初始状态和original相同，但是后续操作不会影响到初始状态，则需要clone方法：

**clone**只是Object的一个protected方法，这意味着你的代码不能直接调用这个方法。这意味着，只有用Employee类能克隆Employee对象。

如果直接克隆这个类的一个对象，就会出现下图中的现象：


这种现象被称为浅拷贝，默认的克隆对象只克隆了基本类型数据，并没有克隆对象内的引用对象。因此，如果想要克隆对象的同时也要克隆对象的所有子对象，就需要重新定义clone方法实现深拷贝。

对于每个类，需要确定；

  1. 默认的clone方法是否满足需求；
  2. 是否可以在可变的子对象上调用clone来修改默认的clone一个方法；
  3. 是否不该用clone。

实际上第3个选项是默认选项。 如果选择第1项或第2项，类必须:

  1. 实现 Cloneable 接口;
  2. 重新定义 clone 方法， 并指定 public 访问修饰符。

注意：Cloneable接口是Java提供的一组标记接口 (tagging interface)之一。标记接口不包含任何方法，唯一的作用是在类型查询中使用instanceof。

即使clone的默认实现能够满足要求，还是需要实现Cloneable接口，将clone重新定义为public， 再调用super.clone().
